2017 April's Fool

For this year April 1st, I coded a fake Algolia API client in CSS.

It is not a real API client, but a carefully crafted CSS file that gives the illusion of search. There is no call to the API at all, everything is handled through CSS selectors (using and abusing input[value='xxx'] selectors).

I took the list of all Algolia employees (as displayed on the website), wrote a simple Ruby parser to generate all the n-grams, and generated a CSS file from that.

I've tried to replicate as many features from the engine as I could, but was quickly limited by the size of the generated CSS file (generating all potential typos quickly exploded the filesize).

You'll find below several versions with an increasing number of features but also increased file size. Let me know which one is the sweet spot for you.

Keep in mind that this is not to be used in production. It's an April's fool, but is also a nice playground for educational content and explain how search works.

#1: Basic search (CSS Filesize: 800Kb)

  • Fun fact on hover of results
  • Highlight
  • Case insensitive search
  • Search in first name or last name (hits in first name returned first)
  • Ranking by arrival date
  • Handles accented characters (try Clement or Clément)
  • Handles composed words (try Marie or Laure)
  • Handles synonyms (try rcs, pizza or flexbox)

#2: Facetting (CSS Filesize: 2Mb)

  • Same features as #1
  • List of teams, along with count
  • Facet count updated as-you-type
  • Facets ordered by count
  • One facet can be selected and it will update the results
  • Can unselect a facet

#3: Typo-tolerance (CSS Filesize: 2Mb)

  • Same features as #1 (no facetting)
  • Handles typos with one missing letter (try niolas or jnas)
  • Still correctly highlight the typos
  • ⚠ UI is slightly slower

#4: Facetting and typo-tolerance (CSS Filesize: 5.7Mb)

  • Same features as #1, #2 and #3
  • ⚠ UI is sluggish
